# 导读与配置

本文稿是 `buaa` Markdown 论文模板的**最小可编译演示**：不展开真实研究内容，只说明如何配置元数据，以及（下一章）如何书写引用、图表、公式等语法。写作真实论文时，用自己的章节替换本演示正文，保留 `00-meta.md` 与后置命令文件即可。\footnote{封面姓名、题目、摘要均为演示占位，请按学院要求改写。}

编译入口：

```bash
./buaa/scripts/build.sh
```

## 文件怎么排

| 文件 | 作用 |
| ---- | ---- |
| `00-meta.md` | 类选项 + 封面/摘要元数据（必改） |
| `01-导读与配置.md` | 本章：配置说明 |
| `02-语法示例.md` | 正文语法样例（可对照改写） |
| `90`–`95` | 结论 / 参考文献 / 附录 / 成果 / 致谢 / 作者简介 |

`buaa/scripts/build.sh` 会收集 `chapters/` 下全部 `*.md`，按**文件名排序**后交给 Pandoc，因此序号即章节顺序。

### 如何定义 / 添加新章节

1. **新建文件**：在 `chapters/` 下增加 `NN-章节名.md`。正文用 `01`–`89`，后置用 `90+`；建议两位数前缀（`03`、`10`），保证字典序与阅读顺序一致。
2. **写标题**：文件第一行用一个一级标题 `# 章名`（对应 LaTeX 的「章」）。其下用 `##` / `###` 作节、条。
3. **插入位置**：需要插在两章之间时，给新文件一个落在两者之间的前缀即可（例如已有 `01`、`03` 时新建 `02-文献综述.md`）；也可整体重编号。
4. **无需改脚本 / `pandoc.yaml`**：只要文件在 `chapters/*.md` 且排序正确，下次编译自动收录。
5. **后置章**：结论等仍用专用命令开头（如 `\summary`），不要只写 `# 结论` 代替 `\summary`，否则页眉/目录体例可能不对。附录先写 `\appendix`，再写 `# 附录标题`。
6. **删除演示章**：正式写作时可删掉 `01-导读与配置.md`、`02-语法示例.md`，换成自己的 `01-引言.md`、`02-文献综述.md` 等；务必保留 `00-meta.md`。

最小正文示例：

```markdown
# 文献综述

## 国外研究现状

……正文……[@jensen1976]
```

## 模板元数据配置（`chapters/00-meta.md`）

论文级配置集中在 `chapters/00-meta.md` 的 YAML 区：上半部分是 Pandoc / 文档类开关，下半部分 `header-includes` 里是封面与摘要所需的 `\Command{…}`。改真实论文时优先改此文件，不必改 `buaa.cls`。

### Pandoc 文档级字段

Table: `00-meta.md` 顶层 YAML 字段说明 {#tab:meta-yaml}

| 字段                | 取值 / 示例               | 含义                      |
| ----------------- | --------------------- | ----------------------- |
| `documentclass`   | `buaa`                | 使用本仓库模块化论文类             |
| `bibliography`    | `./references.bib`    | 文献库路径（相对论文根目录）          |
| `csl`             | `./buaa/gb-t-....csl` | GB/T 7714-2015 顺序编码上标体例 |
| `indent`          | `true` / `false`      | 是否段首缩进                  |
| `link-citations`  | `true` / `false`      | 文内引用是否做成可点击链接           |
| `numbersections`  | `true` / `false`      | 是否对节标题编号（与类选项共同作用）      |
| `classoption`     | 字符串列表                 | 传给 `buaa` 的类选项，见下表      |
| `header-includes` | LaTeX 命令列表            | 封面元数据、摘要、TikZ 宏包等       |

`from`、`pdf-engine`、Lua 过滤器、`citeproc` 等**构建执行项**写在 `buaa/pandoc.yaml`，不要在 `00-meta` 里再开一次 `citeproc`，否则参考文献会重复。

### 类选项 `classoption`

每组通常只选一个；大小写别名（如 `Professional`）与小写等价。未写出的组使用类内默认值。

Table: `classoption` 分组、可选值与含义 {#tab:meta-classoption}

| 分组     | 可选值                                                                                                                     | 含义                                                        | 本演示取值          |
| ------ | ----------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------- | -------------- |
| 报告剖面   | `thesis`、`coursework`、`generic`                                                                                         | 选择封面/目录/后置命令包；学位论文用 `thesis`                              | `thesis`       |
| 学位类型   | `master`、`professional`、`doctor`、`prodoctor`                                                                            | 学硕 / 专硕 / 学博 / 专博；影响封面题头与页眉                               | `professional` |
| 密级     | `public`、`privacy`；`secret`/`secret3`/`secret5`/`secret10`/`secret*`；`classified` 及 `3/5/10/*`；`topsecret` 及 `3/5/10/*` | 封面密级；`secret` 默认按 3 年，`classified`/`topsecret` 默认按 5/10 年 | `public`       |
| 输出形态   | `library`、`print`                                                                                                       | `library` 电子/送交馆藏版式；`print` 在主要分期间加空白背面页                  | `library`      |
| 操作系统提示 | `win`、`linux`、`mac`                                                                                                     | 历史字体宏包提示；本模板以 `buaa/font/` 捆绑字体为准                         | `mac`          |
| 题名长短   | `short`、`long`                                                                                                          | 题名过长时上移封面信息块                                              | `short`        |
| 学科门类版式 | `STEM`、`HSS`                                                                                                            | 理工科小数点式节号 / 人文社科式节号                                       | `STEM`         |
| 字体集    | `fontset=none`（亦可写 `fontset=fandol` 等并传给 ctex）                                                                          | 推荐 `none`，由 `buaa/font/setup.tex` 加载捆绑字体                  | `fontset=none` |

学位为 `master` 或 `doctor`（学术型）时，还必须给出 `\Discipline` 与 `\Direction`；`professional` / `prodoctor` 不强制。

| 命令 | 参数形式 | 必填 | 含义 |
| ---- | -------- | ---- | ---- |

| `\Subtitle{中}{en}` | 副标题 | 否 | |
| `\Author{中}{en}` | 作者 | 是 | |
| `\StudentID{…}` | 学号 | 是 | |
| `\Department{中}{en?}` | 学院 | 是 | 英文可省略并尝试自动翻译 |
| `\Degree{en}` | 英文学位名 | 是 | 如 `Master of Business Administration` |
| `\Major{中}` | 专业 | 是 | |
| `\Feild{中}` | 研究方向 | 是 | 拼写为 `Feild`（与官方类一致） |
| `\Discipline{中}` | 一级学科 | 学硕/学博必填 | |
| `\Direction{中}` | 学科方向 | 学硕/学博必填 | |
| `\Branch{…}` | 门类简称 | 否 | 如「工学」「工商管理」 |
| `\Tutor{中}{en}{职称}` | 导师 | 是 | |

| `\SpecialProg{中}` | 专项计划 | 否 | |
| `\CLC{…}` | 中图分类号 | 否 | |
| `\Abstract{中}{en}` | 摘要 | 建议 | |
| `\Keyword{中}{en}` | 关键词 | 建议 | |
| `\DateEnroll{m}{d}{y}` 等 | 入学/毕业/提交/答辩 | 是 | 另有 `\DateGraduate`、`\DateSubmit`、`\DateDefence` |
| `\Listfigtab{on\|off}` | 开关 | 否 | 图/表清单 |
| `\Signs{…}` / `\Abbreviations{…}` | 表格体 | 否 | 符号表 / 缩略语 |

可在同一列表加载 `\usepackage{tikz}` 与 `\usetikzlibrary{…}`。页边距由类固定，请勿调用 `\geometry`。
